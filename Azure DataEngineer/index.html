Azure Data Factory (ADF)
Cloud-based ETL/ELT orchestration service — the backbone of Azure data pipelines

Managed cloud ETL/ELT service for complex hybrid data integration. Orchestrates data movement and transformation across 90+ cloud and on-premises sources at enterprise scale without managing any infrastructure.

🧪 Hands-on Labs

Lab 5Review the ADF pricing calculator — estimate cost for a simple daily copy pipeline

Control Plane (Azure-hosted UI & APIs for authoring/monitoring) + Data Plane (Integration Runtimes executing data movement). Actual data never passes through the Control Plane — it moves directly between source and sink via the IR.

🧪 Hands-on Labs
Lab 1Draw and label the ADF architecture diagram — Control Plane vs Data Plane

Lab 5Compare the activity run input/output JSON to confirm data did not pass through ADF control plane

Three main tabs: Author (create pipelines, datasets, linked services) · Monitor (track all pipeline, activity, and trigger runs) · Manage (linked services, IRs, triggers, global parameters, credentials).

🧪 Hands-on Labs

Seven core building blocks: Pipelines · Activities · Linked Services · Datasets · Triggers · Data Flows · Integration Runtimes. Understanding how these relate to each other is the foundation of every ADF solution.
